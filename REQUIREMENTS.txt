5. Optional language switch

Add a language option for user-facing CLI output:

```bash

gt-dataslicer --lang en-US ...
```

or, if global Typer options are awkward in the current CLI structure, add the smallest maintainable equivalent that works consistently across commands.

Default:
- `pt-BR`

Supported:
- `pt-BR`
- `en-US`

Behavior:
- `pt-BR` shows Portuguese CLI/help/success/error text.
- `en-US` shows English CLI/help/success/error text.
- Invalid language values should fail with a clear validation error.

Prefer a small translation helper/module over scattering conditional strings throughout the code.
